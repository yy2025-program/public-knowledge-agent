🔑 GitHub Personal Access Token 设置指南

步骤 1: 访问 GitHub 设置页面

1. 登录你的 GitHub 账号

2. 点击右上角头像，选择 "Settings"

3. 在左侧菜单中找到 "Developer settings"

4. 点击 "Personal access tokens" → "Tokens (classic)"

或直接访问：https://github.com/settings/tokens

步骤 2: 创建新的 Token

1. 点击 "Generate new token" → "Generate new token (classic)"

2. 填写 Token 描述，例如：AI Knowledge Agent

3. 设置过期时间（建议选择 "No expiration" 或较长时间）

步骤 3: 选择权限范围

为了使用 GitHub Models API，你需要选择以下权限：

✅ repo (完整仓库访问权限)
✅ user (用户信息访问权限)
✅ read:org (组织读取权限)
✅ workflow (工作流权限)
⚠️ 重要： 确保选择了 repo 权限，这是访问 GitHub Models API 的必要权限。

步骤 4: 生成并复制 Token

1. 点击页面底部的 "Generate token" 按钮

2. 立即复制生成的 Token（只会显示一次！）

3. Token 格式类似：ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

步骤 5: 在 AI 配置页面使用 Token

1. 返回 AI Knowledge Agent 的 AI 配置页面

2. 将复制的 Token 粘贴到 "GitHub Personal Access Token" 输入框

3. 选择你想使用的 AI 模型

4. 点击 "测试连接" 验证 Token 是否有效

5. 如果测试成功，点击 "保存配置"

✅ 成功配置后你将获得：

  • 免费使用 GPT-4o、Claude 3.5 Sonnet 等顶级 AI 模型
  • 每月大量的免费 API 调用额度
  • 更智能、更准确的问答体验

🔒 安全提醒：

  • 不要将 Token 分享给他人
  • 不要在公开场所展示 Token
  • 如果 Token 泄露，立即在 GitHub 设置中删除并重新生成
  • 定期检查 Token 的使用情况

🔧 常见问题解决：

Q: 提示 "401 Unauthorized" 错误？

A: Token 可能无效或已过期，请重新生成 Token。

Q: 提示 "403 Forbidden" 错误？

A: Token 权限不足，请确保选择了 repo 权限。

Q: 提示 "429 Too Many Requests" 错误？

A: API 调用频率超限，请稍后再试。
